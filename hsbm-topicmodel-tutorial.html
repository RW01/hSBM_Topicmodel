Make a bipartite word-document network in graph-tool¶

We represent the occurrences of words in documents as an undirected, bipartite, multigraph.

Given d=1,...,D documents and w=1,...,V word-types (vocabulary) and N words in total, we obtain a graph with n=1,...,D+V nodes and N edges. The first D nodes (n=1,...,D) will be document-nodes ('kind'=0) and the remaining V nodes (n=D+1,...,D+V) will be word-nodes ('kind'=1).

In [4]:

Doing Inference¶

Find groups (or communities) in the network.

Since we have a bipartite network, the $V$ word-nodes will be clustered into $Bw$ word-groups, and the $D$ document-nodes will be clustered into $Bd$ document-groups, such that we get $B = Bd+Bw$ groups in total.

Furthermore, we will fit a hierarchical model, such that we obtain this group structure on $L$ levels of increasing granularity --- until we end up with 1 trivial group containing all nodes.

Note that the number of levels in the hierarchy, $L$, and the number of groups $Bd$ and $Bd$ (on each level of the hierarchy) are determined automatically.

Group membership distributions¶

Document group membership¶

p_td_d is an array of shape B_d x D

  • the first index refers to the doc-group
  • the second index refers to the documents (see 'docs')

Word group membership¶

p_tw_w is an array of shape B_w x V

  • the first index refers to the word-group
  • the second index refers to the words (see 'words')

Note that we are considering a non-overlapping stochastic blockmodel. Therefore, each node (word or document) can belongs to one and only one group.
